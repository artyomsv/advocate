# Plan 25 — Forms (product + legend CRUD from the UI)

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development.

**Goal:** Stop requiring SQL / curl for product and legend management. Dashboard gets full edit surfaces for the product brief, a multi-step legend-create wizard, legend edit, and legend-account creation. After this plan, the full "set up a campaign" flow runs from the browser. Tag `plan25-complete`.

**Prerequisites:** Plans 18-20 complete. Backend routes for `/products`, `/legends`, `/legend-accounts` already exist (shipped in Plans 08, 08.5, 19).

## Architecture

- **Form library:** `react-hook-form` + `@hookform/resolvers/zod`. Standard pattern; tree-shakeable; works with the glass inputs we write.
- **Validation:** client side has light "required / length" checks via zod schemas mirroring backend shape; heavy validation still runs server-side. Error responses map into form errors via a small adapter.
- **Two UI patterns:**
  - **Drawer** — slide-in from right for edits (Product brief, Legend edit, Add account). ~450 px wide, covers part of the page.
  - **Dialog** — centered modal for confirmations + small prompts.
- **Wizard:** the only multi-step form is legend create (~40 fields). Separate route `/legends/new` with step indicator, back/next, save draft to localStorage between refreshes.

`Input`, `Textarea`, `Label`, `Field` are thin wrappers around native elements with glass-aware styling (bg transparent, border `var(--glass-border)`, focus ring `var(--accent-ring)`).

`Drawer` renders a portal with a semi-transparent backdrop + a right-anchored glass panel that slides in via CSS transform (200ms). Close on backdrop click or ESC. Not Radix — hand-rolled is simpler for a single-variant drawer.

`Dialog` wraps Radix's primitive — centered modal with the same glass treatment.

`ListInput` takes `{ value: string[], onChange, placeholder }`. Each row = glass input + trash button. Add button at bottom. Reorder via up/down arrows (drag-drop deferred — keyboard-accessible + good enough).

### Task 3 — Legend create wizard

**Files:**
- `packages/dashboard/src/routes/pages/LegendNew.tsx` (new — 4-step wizard)
- `packages/dashboard/src/components/legends/steps/` — one file per step

  - `PersonalityStep.tsx` — Big Five sliders, techSavviness slider, typing style accordion
  - `ProfessionalStep.tsx` — occupation, company, industry, yearsExperience, education, hobbies (ListInput), expertiseAreas (ListInput), lifeDetails (marital status, children)
  - `ProductRelationshipStep.tsx` — discoveryStory, usageDuration, satisfactionLevel, useCase, complaints (ListInput), alternativesConsidered (ListInput) + maturity dropdown
  - (Preview step is the fifth — just a read-only summary with "Create" button)
- `packages/dashboard/src/hooks/useLegendMutations.ts` (new — `useCreateLegend`)
- `packages/dashboard/src/routes/router.tsx` — register `/legends/new`
- `packages/dashboard/src/routes/pages/Legends.tsx` — add "+ New legend" button linking to `/legends/new`

Use react-hook-form with a single form instance spanning all 5 steps (fields simply not shown on their step). This lets validation propagate and preview step can render everything.

Step indicator across the top:
```
╭─ [ Identity ] — Personality — Professional — Product — Preview ─╮
```
Active step orange; done steps muted.

Save draft to localStorage (key `mynah-legend-draft`) on every field change, debounced 500 ms. "Discard draft" button clears + resets. On successful create, clear draft + navigate to `/legends`.

Submit calls `POST /legends` with the flat form values; server-side zod does the heavy validation.
