# Plan 18 — Campaign Dashboard + Agents Org/Activity (design spec)

> Implementation plan lives in `docs/plans/2026-04-17-18-campaign-ui.md` (to be written after you sign off on this doc). Design decisions here are the input to that plan.

**Locked-in choices**
- Products UX: **P3 — campaign dashboard with top-of-app product switcher**
- Agents UX: **A2 org chart + A3 activity stream** (two tabs on the same page)
- Aesthetic: modern glass, no emojis, lucide icons
- Dark theme: black + orange; light theme: grey + orange
- Theme toggle in topbar

---

## 1. Design tokens

### Orange anchor

A single accent oklch, tuned to read well on both black and grey:

```
--accent:         oklch(0.70 0.19 45)    /* ≈ #ff7a1a — warm, saturated, high-contrast */
--accent-hover:   oklch(0.75 0.19 45)
--accent-muted:   oklch(0.70 0.19 45 / 0.15)   /* translucent for glass fills */
--accent-ring:    oklch(0.70 0.19 45 / 0.45)   /* focus rings */
```

### Dark theme (default)

```
--bg:             #050505                 /* near-black, not pure to reduce OLED ghosting */
--bg-elevated:    #0d0d0d                 /* cards' resting surface under the glass layer */
--fg:             #f5f5f4                 /* neutral-100 */
--fg-muted:       #a1a1aa                 /* zinc-400 */
--fg-subtle:      #52525b                 /* zinc-600 */
--border:         rgb(255 255 255 / 0.08)
--border-strong:  rgb(255 255 255 / 0.14)

--glass:          rgb(255 255 255 / 0.04)  /* card fill before blur */
--glass-hover:    rgb(255 255 255 / 0.06)
--glass-blur:     20px
--glass-border:   rgb(255 255 255 / 0.08)
```

### Light theme

```
--bg:             #f5f5f4                 /* stone-100, warmer than pure grey */
--bg-elevated:    #fafaf9                 /* stone-50 */
--fg:             #18181b                 /* zinc-900 */
--fg-muted:       #52525b                 /* zinc-600 */
--fg-subtle:      #a1a1aa                 /* zinc-400 */
--border:         rgb(0 0 0 / 0.08)
--border-strong:  rgb(0 0 0 / 0.14)

--glass:          rgb(255 255 255 / 0.55)
--glass-hover:    rgb(255 255 255 / 0.70)
--glass-blur:     20px
--glass-border:   rgb(0 0 0 / 0.06)
```

### Glass card recipe

```css
.glass {
  background: var(--glass);
  backdrop-filter: blur(var(--glass-blur)) saturate(140%);
  -webkit-backdrop-filter: blur(var(--glass-blur)) saturate(140%);
  border: 1px solid var(--glass-border);
  border-radius: 14px;
  box-shadow:
    inset 0 1px 0 rgb(255 255 255 / 0.06),
    0 8px 24px -12px rgb(0 0 0 / 0.3);
}
```

Subtle inner-top highlight + outer drop — the two together sell the "pane of glass" feel at small sizes.

### Background behind everything

Dark theme: `#050505` base plus two oversized, blurred orange radial glows anchored off-screen top-left and bottom-right. Low opacity (`--accent-muted`). They show through the glass as faint warmth — not full bloom.

Light theme: `#f5f5f4` base plus the same two radial glows at higher opacity and lower saturation so they read as warm tint, not orange smears.

### Typography

System stack stays (Inter/San Francisco/Segoe UI). Three weights only: 400 body, 500 emphasis, 600 headings. No 700+ — looks dated next to the glass surfaces.

### Icons

`lucide-react` throughout. Icon sizes: 14px (in badges), 16px (in buttons + nav), 20px (in card headers). Stroke width 1.75 everywhere for consistency.

Specific picks:
- Product switcher: `ChevronsUpDown`
- Theme toggle: `Sun` / `Moon` pair
- Nav items: `LayoutDashboard`, `Inbox` (queue), `Users` (legends), `Zap` (LLM), `Bot` (agents)
- Approve/Reject: `Check`, `X`
- Status dots: solid circles in semantic colours (no emoji)

```

│  [ ≡ ] [ Fairy Book Store ⌄ ]                         [ ☀ ] owner [ ⏻ ] │   ← Topbar (glass, sticky)

│         │                                                                │
╰─────────┴────────────────────────────────────────────────────────────────╯

```

**Topbar (left → right):**
- Sidebar collapse button (lucide `Menu`) — replaces current `☰` emoji
- Product switcher — lucide `ChevronsUpDown` with the selected product name. Click → dropdown list of products with the active one highlighted in accent.
- Spacer
- Theme toggle (lucide `Sun` ↔ `Moon`)
- Username in muted text
- Logout button (lucide `Power`, red on hover; click signoutRedirect)

**Sidebar nav:**
- `LayoutDashboard` Home (collapses to just the icon)
- `Inbox` Queue
- `Bot` Agents — NEW
- `Users` Legends
- `Zap` LLM

Active item: orange left-edge bar + accent-muted background. Hover: subtle glass lift.

**Selected product is Zustand state**, persisted to localStorage. All pages read from `useSelectedProduct()`. On first visit with no selection, auto-pick the first product or show an empty state prompting "create a product first."

**Data for Activity tab (thin version until Plan 11.5):**
- For each `content_plan`, we show what we CAN reconstruct from what's persisted:
  - `status` (decision outcome)
  - `qualityScore` jsonb (QualityGate's output)
  - `generatedContent` length + `reviewedBy` (Writer's provider)
  - `rejectionReason` if any
  - `costMillicents` aggregated across all agents that touched the plan (from `llm_usage` where we can correlate)
- What we CAN'T show yet: the per-message handoffs between agents. Each card shows "via this path" as best-effort inference from the final state.

After Plan 11.5 persists the inter-agent message log, the same page gets richer: each agent's actual utterance, timing, and tokens.

## 5. Backend endpoints needed

1. `GET /products/:id/dashboard`
   ```json
   {
     "legendCount": 1,
     "queueCount": 1,
     "scheduleCount": 0,
     "costThisMonthCents": 5,
     "knowledge": { "valueProps": [...], "painPoints": [...], ... }
   }
   ```

2. `GET /products/:id/activity?limit=20`
   Recent events — content_plan status changes, legend creates, schedule registrations.

3. `GET /agents/status?productId=...`
   ```json
   [
     { "agentId": "strategist",     "status": "ready", "lastRunAt": "...", "runsToday": 3, "costMillicentsToday": 3, "provider": "anthropic" },
     { "agentId": "content-writer", ... },
     ...
   ]
   ```

4. `GET /agents/activity?productId=...&limit=20`
   List of content_plans with derived per-agent contributions (see Tab 2 above).

All behind `app.authenticate`.
